Question Approval

Here you'll find information about how to approve questions, as well as what you should be doing when you approve a question.

How do I approve questions?

  1. Get access from the admin (Michael). You've already done this since you can view this page.
  2. Familiarise yourself with the guidelines for approving questions. These will gradually become more complete as Michael gets his act together.
  3. Assign yourself to a group of questions.
    1. Click "Pick questions to approve" at the bottom of the page.
    2. Choose a block from the list.
    3. A page will appear with a list of teaching activities from which you can choose. Assign yourself to approve some activities of your choice. You can see which activities are already assigned for approval.
  4. Once you have activities assigned for approval, click on the second button at the bottom of this page to start approving questions.
  5. Read through the question, the options, the answer and the explanation. Decide if the question meets the guidelines listed below.
    • Only delete a question if it is a duplicate of another question
    • If the question meets the guidelines click "Approve question" under the admin functions on the right hand side of the page (or at the bottom of the page if your screen is too small).
    • If the question blatantly does not meet the guidelines, skip it for now.
    • If the question contains a small error like a spelling mistake, or something which doesn't involve a change in content, you can edit it yourself, providing a reason, and then approve the question. These will usually be things which don't follow the style guidelines.
    • If you aren't sure then click "Flag for review" and explain why you can't decide.
  6. You can approve, edit, delete, flag or skip a question based on the rules above.
  7. Repeat steps 5 and 6 ad infinitum (or ad nauseam, depending on how you see it).

Some extra things to keep in mind when approving questions.

There is a button on the question approval page which says "Edit attributes". This adds extra notes to questions which may be required in certain cases..

Some questions may require special formatting to be displayed. If this is the case, then set the relevant attribute. This is mentioned below for superscript or subscript, but applies to anything which isn't displaying very nicely. Michael will look at these and fix them up if needed.

Exceptional questions may be worth sending to the Faculty or used in a model quiz. Generally, any good question can be sent to the Faculty for review. The best of the best questions will be put into a model quiz for students to test themselves. You can use your judgement to decide if a question satisfies either of these criteria and set the attribute as required.

Guidelines

Content

Michael is receiving advice from the Assessment Unit before he completes this section.

Questions must have one, and only one, most correct answer. Whilst the other options may contain correct information, they must not be completely correct. There must be one question which is obviously more correct than the others.

In general, the content for a question should only be related to the teaching activity for which it was written. PBLs are allowed to contain questions which span multiple teaching activities for that week.

Explanations should contain enough information to be able to figure out the answer with only a basic science background. Users must write an explanation for the answer, and this is enforced by the system. They should also (but don't have to) explain why the other answers are incorrect.

{% comment %}

Explanations must not refer to powerpoint slides or make any other reference to lecture resources (such as notes). We cannot guarantee over a number of years that the old lecture resources will remain available, so the explanation must not rely on them. If the answer is an obvious, non-contentious fact contained in any reputable textbook, then the explanation should simply say so.

{% endcomment %}

Do not spend endless hours fact-checking questions. It is important that the question is correct, but it does not need to be correct down to the last detail. Part of the beauty of MedBank is that students can contribute to the process of improving questions over time. If a question or explanation is correct except for one small detail, approve it. If a question is completely wrong, then it is contrary to the guidelines. A good rule of thumb is to spend five minutes maximum per question

Style

This part of the guide is all about consistency of presentation, as well as ensuring that every single person who answers the question can understand it compeletely and fully. A stylistically correct question should be easily understandable, so that only a person's knowledge of the content is required to comprehend and answer it.

Note that the format for explanations is currently changing. You may encounter explanations which were written before the change in format (where it was a free text field, and not linked to question options). If this is the case, don't worry, just approve the question (if everything else is compatible with the guidelines, of course).

Questions need to be written in grammatically correct English. This includes the correct usage of terms such as your and you're; two, too and to; they're, their and there; and so on. They should also be written with correct spelling and punctuation.

If a question is written in a "complete the statement" style, the answers should make sense. Each option should complete the sentence in an understandable and grammatically correct way. This means that every option should end with a full stop. For example:

The ending of this question
  1. does not make sense.
  2. makes complete sense.
  3. is irrelevant.
  4. is not obvious.
  5. always completes the sentence.
Every option for this question completes the stem in a way which makes sense (ignoring the fact that it does not have an obvious answer). Every option finishes with a full stop, and the stem does not have any punctuation at the end.

Abbreviations should not be used without first being defined. This is non-negotiable, and includes all acronyms for diseases, chemical symbols, drug names, and any other acronym or shorthand notation which is used. The term should be used in full for the first mention, with the abbreviation defined in brackets. For example:

Chronic obstructive pulmonary disease (COPD), sodium (Na), serotonin (5-HT) and acetylcholine (ACh) are all valid definitions of abbreviations. Every following mention can then use COPD, Na, 5-HT or ACh.
Abbreviations should also be consistent with the commonly used abbreviation in the literature.

Please remove any symbols trying to denote superscript or subscript notation. Superscript and subscript are going to be added features in the future. If there is an underscore (_) or a caret (^) used to denote subscript or superscript respectively, please delete it by editing the question. Then, click "Edit Attributes" and check the box about requiring subscript or superscript. For example, 5-HT_1 is not appropriate (representing 5-HT1), please change it to 5-HT1.
